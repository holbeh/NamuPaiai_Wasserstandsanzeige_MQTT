134  * -D UMM_POISON :
135  *
136  * Enables heap poisoning: add predefined value (poison) before and after each
137  * allocation, and check before each heap operation that no poison is
138  * corrupted.
139  *
140  * Other than the poison itself, we need to store exact user-requested length
141  * for each buffer, so that overrun by just 1 byte will be always noticed.
142  *
143  * Customizations:
144  *
145  * UMM_POISON_SIZE_BEFORE:
146  * Number of poison bytes before each block, e.g. 2
147  * UMM_POISON_SIZE_AFTER:
148  * Number of poison bytes after each block e.g. 2
149  * UMM_POISONED_BLOCK_LEN_TYPE
150  * Type of the exact buffer length, e.g. `short`
151  *
152  * NOTE: each allocated buffer is aligned by 4 bytes. But when poisoning is
153  * enabled, actual pointer returned to user is shifted by
154  * `(sizeof(UMM_POISONED_BLOCK_LEN_TYPE) + UMM_POISON_SIZE_BEFORE)`.
