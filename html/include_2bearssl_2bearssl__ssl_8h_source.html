2049 /*
2050  * There are four I/O operations, each identified by a symbolic name:
2051  *
2052  * sendapp inject application data in the engine
2053  * recvapp retrieving application data from the engine
2054  * sendrec sending records on the transport medium
2055  * recvrec receiving records from the transport medium
2056  *
2057  * Terminology works thus: in a layered model where the SSL engine sits
2058  * between the application and the network, "send" designates operations
2059  * where bytes flow from application to network, and "recv" for the
2060  * reverse operation. Application data (the plaintext that is to be
2061  * conveyed through SSL) is "app", while encrypted records are "rec".
2062  * Note that from the SSL engine point of view, "sendapp" and "recvrec"
2063  * designate bytes that enter the engine ("inject" operation), while
2064  * "recvapp" and "sendrec" designate bytes that exit the engine
2065  * ("extract" operation).
2066  *
2067  * For the operation 'xxx', two functions are defined:
2068  *
2069  * br_ssl_engine_xxx_buf
2070  * Returns a pointer and length to the buffer to use for that
2071  * operation. '*len' is set to the number of bytes that may be read
2072  * from the buffer (extract operation) or written to the buffer
2073  * (inject operation). If no byte may be exchanged for that operation
2074  * at that point, then '*len' is set to zero, and NULL is returned.
2075  * The engine state is unmodified by this call.
2076  *
2077  * br_ssl_engine_xxx_ack
2078  * Informs the engine that 'len' bytes have been read from the buffer
2079  * (extract operation) or written to the buffer (inject operation).
2080  * The 'len' value MUST NOT be zero. The 'len' value MUST NOT exceed
2081  * that which was obtained from a preceding br_ssl_engine_xxx_buf()
2082  * call.
2083  */

3931 /*
3932  * Context for the simplified I/O context. The transport medium is accessed
3933  * through the low_read() and low_write() callback functions, each with
3934  * its own opaque context pointer.
3935  *
3936  * low_read() read some bytes, at most 'len' bytes, into data[]. The
3937  * returned value is the number of read bytes, or -1 on error.
3938  * The 'len' parameter is guaranteed never to exceed 20000,
3939  * so the length always fits in an 'int' on all platforms.
3940  *
3941  * low_write() write up to 'len' bytes, to be read from data[]. The
3942  * returned value is the number of written bytes, or -1 on
3943  * error. The 'len' parameter is guaranteed never to exceed
3944  * 20000, so the length always fits in an 'int' on all
3945  * parameters.
3946  *
3947  * A socket closure (if the transport medium is a socket) should be reported
3948  * as an error (-1). The callbacks shall endeavour to block until at least

3950  * acceptable, but this normally leads to the callback being immediately
3951  * called again, so the callback should at least always try to block for
3952  * some time if no I/O can take place.

3954  * The SSL engine naturally applies some buffering, so the callbacks need
3955  * not apply buffers of their own.
